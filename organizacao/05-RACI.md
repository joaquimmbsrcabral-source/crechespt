# Matriz RACI — Creches.app (v3, Jul 2026)
*Fonte canónica. **R** = Responsável (faz) · **A** = Aprova (decide) · **C** = Consultado · **I** = Informado*

Regras de leitura: cada atividade tem exatamente **um R** e **um A** (podem coincidir). Os freelancers nunca têm A — **quem contrata, gere e avalia freelancers é a COO** (com orçamento aprovado pelo Fundador). "FL" = freelancer da área.

## Produto e tecnologia

| Atividade | Fundador | COO | Dev | FL |
|---|:-:|:-:|:-:|:-:|
| Visão e roadmap de produto | **R+A** | C | C | – |
| Priorização de sprint (quinzenal) | **A** | C | **R** | – |
| Desenvolvimento, deploys, incidentes | A | I | **R** | – |
| Revisão de mudanças críticas (regras/billing) | **R+A** | – | C | – |
| Pipeline de dados (dataset, fichas) | A | C | **R** | – |
| Segurança e RGPD técnico | A | C | **R** | – |
| QA de novas features (olhar de utilizador) | I | **R** | C | – |

## Escalação (quando não está claro)

1. Operacional do dia-a-dia → decide a **COO**
2. Toca no produto/código → decide o **Dev** dentro do sprint; fora do sprint → Fundador
3. Toca em dinheiro >€500, marca, FJM ou princípios (mapa não se compra, gratuito para famílias) → **Fundador, sempre**
4. Dúvida entre dois donos → weekly de 6ª feira; urgente → chamada ao Fundador
